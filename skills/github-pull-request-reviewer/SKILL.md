# Pull Request 리뷰

## 분석
- 프롬프트에 PR 주소(URL)가 없다면 먼저 사용자에게 대상 PR 링크를 요청해 확인한다.
- gh CLI로 PR을 조회해 변경 파일과 내용을 확인한다. 예: `gh pr view <URL> --files`, `gh pr diff <URL>`로 파일/차이를 확인하고, 필요 시 `gh pr checkout <URL>`으로 로컬 재현한다.
- PR의 변경 파일, 주요 로직, 테스트 수정 여부를 확인한다.
- 기능 영향, 리스크(회귀, 성능, 보안), 누락된 테스트나 문서 필요성을 점검한다.

## 리뷰 작성
- 발견 사항이 없으면 “리뷰 없음”으로 답변하고 종료한다.
- 발견 사항이 있으면 요약(한글)과 함께 필요한 경우 파일/라인 기준으로 코멘트를 남기거나 제안한다.
- 제안이 테스트에 영향을 준다면 필요한 테스트를 명시한다.
