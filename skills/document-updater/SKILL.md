# 문서 최신화 (리비전 기준)

## 목표
- 아키텍쳐 문서 `docs/architecture.md`와 개발 가이드 문서 `docs/development.md`를 최신화한다.

## 작업 규칙
- 문서가 존재하지 않으면 사용자에게 알리고 작업을 건너뛴다.

## 절차
- 문서의 메타 정보에 리비전이 있다면, 그 값은 "현재 문서 내용이 어느 리비전까지 반영되어 있는지"를 의미한다고 해석한다. 이 경우 해당 리비전 이후의 변경 내역을 기준으로 문서를 수정한다.
- 문서의 메타 정보에 리비전이 없다면 사용자에게 기준 리비전을 확인해 입력받는다(예: “기준 리비전/커밋 해시를 알려주세요”). 입력받은 리비전 이후의 변경 내역을 기준으로 문서를 수정한다.
- 기준 리비전 이후 커밋을 확인해 기능 변화, 제거된 코드, 추가된 흐름을 파악한다.
- 각 문서의 목적(아키텍처 개요, 개발 가이드)을 고려하여 내용이 코드와 불일치하는지 점검한다.
- 코드에서 삭제되어 더 이상 유효하지 않은 설명은 조용히 제거한다(“삭제됨” 등의 언급 금지).
- 코드 변경으로 달라진 부분은 설명을 수정하거나 필요한 내용을 추가한다. 문서의 목적에 맞지 않은 내용은 문서에 반영하지 않는다.
- 문서를 수정한 후 문서의 메타 정보를 최신화한다. 기존 메타 정보가 있었다면 리비전은 문서를 수정한 시점의 최신 리비전으로 갱신한다.
- 문서의 메타 정보가 없어 사용자에게 기준 리비전을 입력받아 작업했다면, 문서 수정 후 문서 상단에 메타 정보를 추가한다. 이때 리비전은 문서를 수정한 시점의 최신 리비전을 기록한다.
- 반영할 변경이 없다면 작업을 종료한다.
