# 개발 가이드 문서 작성

## 목표
- `docs/development.md`에 프로젝트 개발에 도움이 되는 특화 정보를 한국어로 정리하라.
- 숙련 개발자 관점에서 기본 설명을 배제하고, 프로젝트에만 해당하는 내용만 기록하라.

## 작성 규칙
- 출력은 순수 Markdown 문서 형식으로 작성하라.
- 문서 제목은 반드시 `# 개발 가이드`로 시작하라.
- 필요 시 항목을 추가하되, 아래 필수 범주는 반드시 포함하라.

## 필수 포함 내용
1. **문서 정보**
   - 문서를 작성한 시점의 revision, 작성일시를 메타 정보로 남긴다. 
2. **빌드/구성 지침**
   - 특정 빌드/구성 단계가 있으면 명확하고 상세하게 설명하라.
3. **프로젝트 구조/구성/스타일/패턴**
   - 디렉터리 구조, 코드 스타일, 주요 디자인 패턴 등을 프로젝트 특화 관점에서 정리하라.
4. **테스트 관련 정보**
   - 테스트 설정 및 실행 방법.
   - 새로운 테스트를 추가하고 실행하는 가이드라인.
   - 간단한 테스트를 실제로 작성하고 실행하는 시연을 포함하라.
   - 개발/디버깅에 유용한 추가 정보를 포함하라.
5. **보안 정보 점검**
   - password/key 등 민감 정보가 직접 포함되어 있는지 점검하고, 발견 시 수정 필요성을 명시하라.

## 검증 요구사항
- 문서에 테스트 예제를 포함하기 전에, 실제로 동작하는지 확인하라.
- 테스트를 실행할 수 없거나 환경 제약이 있으면, 무엇이 막혔는지와 대체 확인 방법을 명시하라.

## 실행 절차 가이드
1. 저장소를 탐색해 빌드/테스트/구성 관련 문서와 스크립트를 확인하라.
2. 실제 테스트 실행을 시도하고, 성공/실패 결과를 기록하라.
3. 프로젝트 구조 및 코드 스타일 단서를 추출해 문서에 요약하라.
4. 보안 점검 결과를 기록하라.
5. `docs/development.md`를 갱신하라.
