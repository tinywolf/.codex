# Git Commit

## 기본 절차
- `git status`로 변경 파일과 상태를 확인하고, 커밋 대상 파일을 결정한다.
- 필요한 파일만 `git add <file>`로 스테이징한다.
- 커밋 메시지는 한국어로 작성하고, 변경 목적과 의도를 명확히 드러내도록 한 줄 제목으로 정리한다. 예: `"API 응답 캐싱으로 대기 시간 단축"` 또는 `"테스트 누락 보완 및 문서 업데이트"`.
- `git commit -m "<커밋 메시지>"`로 커밋한다.
- `git status`로 워킹트리가 깨끗한지 확인한다.

## 추가 팁
- 커밋 전에 필요한 경우 빠르게 확인이 가능한 테스트를 실행해 변경이 안전한지 점검한다.
- 작업 범위가 큰 경우 논리 단위로 나누어 여러 커밋으로 작성한다.
