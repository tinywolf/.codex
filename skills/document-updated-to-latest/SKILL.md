# 문서 최신화 (리비전 기준)

## 목표
- 아키텍쳐 문서 `docs/architecture.md`와 개발 가이드 문서 `docs/development.md`를 최신화한다.

## 작업 규칙
- 문서가 존재하지 않으면 사용자에게 알리고 작업을 건너뛴다.

## 절차
- 기준 리비전이 없다면 먼저 문서에 포함된 메타 정보를 확인한다. 메타 정보에 기준 리비전이 없다면 사용자에게 기준 리비전을 확인해 입력받는다(예: “기준 리비전/커밋 해시를 알려주세요”).
- 기준 리비전 이후 커밋을 확인해 기능 변화, 제거된 코드, 추가된 흐름을 파악한다.
- 각 문서의 목적(아키텍처 개요, 개발 가이드)을 고려하여 내용이 코드와 불일치하는지 점검한다.
- 코드에서 삭제되어 더 이상 유효하지 않은 설명은 조용히 제거한다(“삭제됨” 등의 언급 금지).
- 코드 변경으로 달라진 부분은 설명을 수정하거나 필요한 내용을 추가한다. 문서의 목적에 맞지 않은 내용은 문서에 반영하지 않는다.
- 문서 정보(문서에 반영한 최신 revision, 작성일시)를 최신화한다. 문서 정보가 없다면 문서 상단에 추가한다.
- 반영할 변경이 없다면 작업을 종료한다.
