# TASK 평가 루브릭

## 1. ROI 점수 체계
각 항목을 1~5점으로 평가한다.

- `impact`: 사용자 가치, 비즈니스 효과, 장애/리스크 감소 효과
- `urgency`: 지연 시 손실, 일정 임계성, 외부 의존 일정
- `confidence`: 요구사항 명확성, 구현 가능성, 검증 가능성
- `effort`: 개발/리뷰/테스트/배포까지 총 비용
- `risk`: 변경 실패 가능성, 회귀 위험, 운영 영향

계산식:

`roi_score = (impact * urgency * confidence) / max(effort + risk, 1)`

## 2. ROI 해석 가이드
- `roi_score >= 3.0`: 높은 우선순위. 즉시 수행 후보
- `1.5 <= roi_score < 3.0`: 조건부 수행. 범위 축소 또는 선행 작업 보강
- `roi_score < 1.5`: 낮은 우선순위. 보류/제거 후보

점수는 절대값이 아니라 상대 비교 기준으로 사용한다.

## 3. 작업 타당성 체크리스트
- SPEC 요구사항 ID와 1:1 또는 1:N으로 연결되는가?
- 산출물과 완료조건(DoD)이 명확한가?
- 선행 의존성과 후속 영향이 표시되었는가?
- 테스트/관측/롤백 고려가 포함되는가?

## 4. 누락 항목 탐지 체크리스트
SPEC가 존재하면 아래 범주를 최소 1회 확인한다.

- 기능 구현
- 예외/실패 처리
- 테스트(단위/통합/회귀)
- 데이터 마이그레이션 또는 호환성
- 운영 관측(로그/메트릭/알람)
- 배포 및 롤백 전략

## 5. 분할 품질 체크리스트
- 각 TASK가 독립적으로 완료/검증 가능한가?
- 병렬 개발 시 충돌 가능성이 낮은가?
- 리뷰 가능한 크기(과대/과소 아님)인가?
- 사용자 가치가 전달되는 순서로 배치되었는가?
