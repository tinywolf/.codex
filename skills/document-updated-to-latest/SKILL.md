# 문서 최신화 (리비전 기준)

## 절차
- 프롬프트에 기준 리비전이 없다면 먼저 사용자에게 기준 리비전을 확인해 입력받는다(예: “기준 리비전/커밋 해시를 알려주세요”).
- 입력된 기준 리비전 이후 커밋을 확인해 기능 변화, 제거된 코드, 추가된 흐름을 파악한다.
- 각 문서의 목적(아키텍처 개요, 개발 가이드)을 고려하며 내용이 코드와 불일치하는지 점검한다.
- 코드에서 삭제되어 더 이상 유효하지 않은 설명은 조용히 제거한다(“삭제됨” 등의 언급 금지).
- 코드 변경으로 달라진 부분은 설명을 수정하거나 필요한 내용을 추가한다.
- 반영할 변경이 없다면 작업을 종료한다.
