# Agent 가이드라인

## 코딩
- 클래스에 역할과 책임을 명확하게 알 수 있는 내용을 주석으로 추가한다.
- 복잡한 코드나 핵심 기능에는 구현 목적과 의사 결정 과정을 간략하게 요약해서 주석으로 추가한다.
- 변수는 사용되는 위치 근처에 선언하고 직관적인 이름으로 짓는다.

### YAGNI & Simplicity
- **YAGNI 원칙 준수**: 프롬프트에서 명시적으로 요구되지 않는 한, 예측 가능한 기능, 미래 대비 기능 또는 추상적인 아키텍처를 구현하지 말 것.
- **최소한의 코드 영향**: 가능한 한 적은 코드 라인으로 당면한 문제를 해결하라.
- **추가보다는 리팩토링**: 새로운 유틸리티나 클래스를 추가하기 전에 기존 클래스나 클래스를 간단하게 수정할 수 있는지 확인하라.
- **YAGNI 체크리스트**: Codex 계획 생성 시, 모든 클래스/함수가 알려진 활성 요구 사항에 매핑되는지 확인하라.

## 하위 디렉터리 지침
- 특정 하위 디렉터리의 파일을 작업할 때 해당 경로에 별도 `AGENTS.md`가 있으면 먼저 읽고 따른다.
